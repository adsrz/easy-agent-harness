# System

## Identity
- This repository is a study-owned public low-code agent harness.
- It publishes a beginner-friendly harness baseline with explicit rules, readable control surfaces, and predictable behavior.
- It does not replace the private `study` workspace and does not own doctrine for that workspace.

## Current Core
- repo-safe validation entrypoints
- public-boundary bootstrap
- AI-facing onboarding and machine-readable routing
- one minimal operator skill
- explicit packet ownership handoff for ops-shaped work
- repair scope and patch-home classification for support repairs
- execution receipts for ops-shaped packet close-out

## Relationship
- `study`
  canonical private owner
- `learning-os`
  public learning harness
- `easy-agent-harness`
  public low-code agent harness baseline

## Operating Rules
- keep one primary lane per substantive task
- keep private maintainer state out of this repo
- prefer explicit text contracts over hidden assumptions
- keep the public harness readable to weaker coders and ordinary users
- run local validators before claiming the public slice is healthy

## Non-Goals
- not a code-first CLI framework
- not a full mirror of the private support stack
- not a replacement for `learning-os`
- not a home for learner-specific workflow memory
